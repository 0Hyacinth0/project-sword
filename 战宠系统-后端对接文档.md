## 一、数据库表结构

### 1. 战宠类型表 (pet_types)

```sql
CREATE TABLE pet_types (
    pet_type_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '战宠类型ID',
    name VARCHAR(50) NOT NULL COMMENT '战宠名称',
    element TINYINT NOT NULL COMMENT '属性类型: 1-火, 2-水, 3-风, 4-地, 5-光, 6-暗',
    rarity TINYINT NOT NULL COMMENT '品质: 1-N, 2-R, 3-SR, 4-SSR',
    base_hp INT NOT NULL COMMENT '基础生命',
    base_attack INT NOT NULL COMMENT '基础攻击',
    base_defense INT NOT NULL COMMENT '基础防御',
    base_speed INT NOT NULL COMMENT '基础速度',
    hp_growth DECIMAL(5,2) NOT NULL COMMENT '每级生命成长',
    attack_growth DECIMAL(5,2) NOT NULL COMMENT '每级攻击成长',
    defense_growth DECIMAL(5,2) NOT NULL COMMENT '每级防御成长',
    speed_growth DECIMAL(5,2) NOT NULL COMMENT '每级速度成长',
    evolve_to INT DEFAULT 0 COMMENT '进化目标类型ID, 0为最终形态',
    evolve_level INT DEFAULT 0 COMMENT '进化所需等级, 0为不可进化',
    icon VARCHAR(100) COMMENT '图标资源路径',
    description VARCHAR(500) COMMENT '背景描述',
    INDEX idx_element (element),
    INDEX idx_rarity (rarity)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='战宠类型静态配置表';
```

### 2. 角色战宠表 (character_pets)

```sql
CREATE TABLE character_pets (
    id VARCHAR(36) PRIMARY KEY COMMENT '战宠实例UUID',
    character_id VARCHAR(36) NOT NULL COMMENT '所属角色UUID',
    pet_type_id INT NOT NULL COMMENT '战宠类型ID',
    nickname VARCHAR(20) COMMENT '战宠昵称',
    level INT DEFAULT 1 COMMENT '等级',
    experience INT DEFAULT 0 COMMENT '经验值',
    is_active TINYINT(1) DEFAULT 0 COMMENT '是否出战: 0-否, 1-是',
    skill_slot_1 INT DEFAULT NULL COMMENT '技能槽位1',
    skill_slot_2 INT DEFAULT NULL COMMENT '技能槽位2',
    skill_slot_3 INT DEFAULT NULL COMMENT '技能槽位3',
    equip_armor VARCHAR(36) DEFAULT NULL COMMENT '装备位-护甲',
    equip_accessory VARCHAR(36) DEFAULT NULL COMMENT '装备位-饰品',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '获得时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    FOREIGN KEY (character_id) REFERENCES characters(id) ON DELETE CASCADE,
    FOREIGN KEY (pet_type_id) REFERENCES pet_types(pet_type_id),
    FOREIGN KEY (skill_slot_1) REFERENCES pet_skills(skill_id),
    FOREIGN KEY (skill_slot_2) REFERENCES pet_skills(skill_id),
    FOREIGN KEY (skill_slot_3) REFERENCES pet_skills(skill_id),
    INDEX idx_character (character_id),
    INDEX idx_active (character_id, is_active)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='角色拥有的战宠实例表';
```

> **业务约束**：每个 `character_id` 下最多 3 条记录；同一 `character_id` 下 `is_active = 1` 最多 1 条（需后端代码校验）。

### 3. 战宠技能表 (pet_skills)

```sql
CREATE TABLE pet_skills (
    skill_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '技能ID',
    name VARCHAR(50) NOT NULL COMMENT '技能名称',
    element TINYINT NOT NULL COMMENT '属性类型: 0-无, 1-火, 2-水, 3-风, 4-地, 5-光, 6-暗',
    type TINYINT NOT NULL COMMENT '技能类型: 1-主动攻击, 2-主动辅助, 3-被动',
    power INT DEFAULT 100 COMMENT '技能威力倍率(%)',
    cooldown INT DEFAULT 0 COMMENT '冷却回合数',
    learn_level INT DEFAULT 1 COMMENT '习得所需战宠等级, 0为进化专属',
    pet_type_ids JSON COMMENT '可学习该技能的战宠类型ID列表',
    description VARCHAR(200) COMMENT '技能描述',
    INDEX idx_type (type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='战宠技能静态配置表';
```

### 4. 战宠已学技能表 (pet_learned_skills)

```sql
CREATE TABLE pet_learned_skills (
    id VARCHAR(36) PRIMARY KEY COMMENT '唯一标识UUID',
    pet_id VARCHAR(36) NOT NULL COMMENT '战宠实例UUID',
    skill_id INT NOT NULL COMMENT '已学技能ID',
    learned_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '学习时间',
    FOREIGN KEY (pet_id) REFERENCES character_pets(id) ON DELETE CASCADE,
    FOREIGN KEY (skill_id) REFERENCES pet_skills(skill_id),
    UNIQUE KEY uk_pet_skill (pet_id, skill_id),
    INDEX idx_pet (pet_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='战宠已学技能关联表';
```

### 5. 战宠进化材料配置表 (pet_evolve_items)

```sql
CREATE TABLE pet_evolve_items (
    pet_type_id INT NOT NULL COMMENT '进化前的战宠类型ID',
    item_id INT NOT NULL COMMENT '所需材料ID',
    quantity INT NOT NULL COMMENT '所需数量',
    PRIMARY KEY (pet_type_id, item_id),
    FOREIGN KEY (pet_type_id) REFERENCES pet_types(pet_type_id),
    FOREIGN KEY (item_id) REFERENCES items(item_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='战宠进化材料配置表';
```

## 二、属性计算公式

### 战宠实际属性

```
hp = floor((base_hp + level × hp_growth) × rarity_multiplier + equip_hp)
attack = floor((base_attack + level × attack_growth) × rarity_multiplier + equip_attack)
defense = floor((base_defense + level × defense_growth) × rarity_multiplier + equip_defense)
speed = floor((base_speed + level × speed_growth) × rarity_multiplier + equip_speed)
```

**品质倍率对照：**

| rarity | 倍率 |
|:---:|:---:|
| 1 (N) | 1.0 |
| 2 (R) | 1.2 |
| 3 (SR) | 1.5 |
| 4 (SSR) | 2.0 |

### 主人属性加成（仅出战战宠生效）

```
bonusHp = floor(pet.hp × 0.1 × rarity_multiplier)
bonusAttack = floor(pet.attack × 0.1 × rarity_multiplier)
bonusDefense = floor(pet.defense × 0.1 × rarity_multiplier)
```

## 三、副本掉落规则

| 副本类型 | 掉落概率 | 品质分布 |
|:---|:---:|:---|
| 普通副本 | 5% | N 80%, R 18%, SR 2% |
| 精英副本 | 15% | N 50%, R 35%, SR 13%, SSR 2% |
| 团队副本 Boss | 30% | R 40%, SR 45%, SSR 15% |

> **实现建议**：在副本通关结算逻辑中，根据副本类型计算是否掉落战宠，再根据区域配置决定具体掉落的 `pet_type_id`，最后按品质分布概率确定 `rarity`。

## 五、接口详细规范

### 1. 获取角色战宠列表

**GET `/pet/list`**

**Query 参数：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| `characterId` | `string` | ✅ | 角色 UUID |

**响应字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `data[].id` | `string` | 战宠实例 UUID |
| `data[].petTypeId` | `number` | 战宠类型 ID |
| `data[].nickname` | `string` | 战宠昵称 |
| `data[].level` | `number` | 等级 |
| `data[].experience` | `number` | 经验值 |
| `data[].isActive` | `boolean` | 是否出战 |
| `data[].rarity` | `number` | 品质 1-4 |
| `data[].hp` | `number` | 当前生命 |
| `data[].attack` | `number` | 当前攻击 |
| `data[].defense` | `number` | 当前防御 |
| `data[].speed` | `number` | 当前速度 |
| `data[].skillSlots` | `number[3]` | 技能槽，0 表示空 |
| `data[].equipSlots.armor` | `string | null` | 护甲装备 ID |
| `data[].equipSlots.accessory` | `string | null` | 饰品装备 ID |
| `data[].bonusToOwner` | `object` | 给主人的加成 |

### 2. 获取战宠详情

**GET `/pet/{id}`**

**Path 参数：** `id` - 战宠实例 UUID

**响应额外字段：**

| 字段 | 说明 |
|------|------|
| `learnedSkills` | 已学会的所有技能 ID 数组 |
| `evolveInfo.canEvolve` | 是否满足进化条件 |
| `evolveInfo.evolveToTypeId` | 进化目标类型 ID |
| `evolveInfo.evolveRequiredLevel` | 进化所需等级 |
| `evolveInfo.evolveRequiredItems[]` | 进化材料及持有情况 |

---

### 3. 重命名战宠

**PUT `/pet/rename`**

**Body 参数：**

```json
{
  "petId": "uuid",
  "nickname": "新昵称"
}
```

**校验：** 昵称 2-8 字符；操作者必须是战宠所属角色的用户。

---

### 4. 设置出战战宠

**PUT `/pet/set-active`**

**Body 参数：**

```json
{
  "petId": "uuid"
}
```

**逻辑：**
1. 将该角色下所有战宠 `is_active` 设为 `false`
2. 目标战宠 `is_active` 设为 `true`
3. 若 `petId` 为空字符串或 `null`，则取消出战（全部设为 `false`）

---

### 5. 喂食经验道具

**POST `/pet/feed`**

**Body 参数：**

```json
{
  "petId": "uuid",
  "itemId": 4001,
  "quantity": 3
}
```

**逻辑：**
1. 校验背包中该道具数量 ≥ `quantity`
2. 扣除道具，增加战宠经验
3. 若升级，检查是否触发自动习得技能（`learn_level` 匹配）
4. 返回升级信息和新习得技能

**经验计算：**
- 每个经验道具有固定经验值（需在 `items` 表配置）
- 升级阈值参考角色升级曲线，建议独立配置

---

### 6. 战宠进化

**POST `/pet/evolve`**

**Body 参数：**

```json
{
  "petId": "uuid"
}
```

**逻辑：**
1. 查询战宠当前 `pet_type_id`，获取 `evolve_to` 和 `evolve_level`
2. 校验：`level >= evolve_level`；背包中进化材料充足
3. 扣除材料
4. 更新 `pet_type_id` 为进化形态
5. 重新计算属性，保留 80% 当前经验进度
6. 检查是否习得进化专属技能（`learn_level = 0`）
7. 返回进化后属性和新技能

---

### 7. 装备/替换技能

**PUT `/pet/skill-slot`**

**Body 参数：**

```json
{
  "petId": "uuid",
  "slotIndex": 2,
  "skillId": 2010
}
```

**校验：**
- `slotIndex` 范围 0-2
- `skillId` 必须在 `pet_learned_skills` 中存在（该战宠已学会）
- 不能与其他槽位重复装备同一技能
- `skillId = 0` 表示清空该槽位

---

### 8. 穿戴装备

**PUT `/pet/equip`**

**Body 参数：**

```json
{
  "petId": "uuid",
  "slotType": "armor",
  "equipmentId": "eq-uuid"
}
```

**校验：**
- `slotType` 为 `armor` 或 `accessory`
- 装备在角色背包中
- 装备类型为战宠可用（建议在 `items` 表增加 `pet_usable` 字段）

**逻辑：** 若目标槽位已有装备，先自动卸下回背包，再穿戴新装备。

---

### 9. 卸下装备

**PUT `/pet/unequip`**

**Body 参数：**

```json
{
  "petId": "uuid",
  "slotType": "armor"
}
```

**校验：** 背包有空位接收卸下的装备。

---

### 10. 获取战宠图鉴

**GET `/pet/encyclopedia`**

无参数，返回所有战宠类型静态数据。

**响应字段：**

| 字段 | 说明 |
|------|------|
| `petTypeId` | 战宠类型 ID |
| `name` | 战宠名称 |
| `element` | 属性 1-6 |
| `rarity` | 品质 1-4 |
| `baseHp` ~ `baseSpeed` | 基础属性 |
| `hpGrowth` ~ `speedGrowth` | 成长值 |
| `evolveTo` | 进化目标 |
| `evolveLevel` | 进化等级 |
| `icon` | 图标路径 |
| `owned` | 当前角色是否已拥有 |

> **性能建议**：此接口数据静态，建议 Redis 缓存。

## 七、联调注意事项

1. **战宠数量限制**：后端必须校验每个角色最多 3 只，不能仅依赖前端隐藏创建入口
2. **出战唯一性**：设置出战时，需先清除该角色其他战宠的 `is_active` 状态
3. **进化材料**：需先有 `items` 表，并在其中配置进化材料类型
4. **战宠装备**：需在 `items` 表增加字段标识哪些装备可用于战宠
5. **经验道具**：需在 `items` 表配置每个道具提供的经验值
6. **技能习得**：升级后需自动检查 `pet_skills` 中 `learn_level` 匹配且 `pet_type_ids` 包含当前战宠类型的技能
