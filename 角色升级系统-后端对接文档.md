## 一、核心配置参数

| 参数 | 值 | 说明 |
|:---|:---:|:---|
| `MAX_LEVEL` | 100 | 最大等级上限 |
| `POINTS_PER_LEVEL` | 3 | 每升一级获得的自由属性点数 |
| `BASE_EXP` | 100 | 基础经验值系数 |
| `EXP_MULTIPLIER` | 1.15 | 经验增长系数（指数曲线） |
| `EXP_LINEAR` | 50 | 线性增长系数（避免后期过于陡峭） |

---

## 二、经验公式

### 升级所需经验计算

```
nextExp = floor(BASE_EXP × EXP_MULTIPLIER^level + EXP_LINEAR × level)
```

**Java 实现：**

```java
/**
 * 计算从当前等级升到下一级所需的总经验值
 * @param level 当前等级
 * @return 升级所需经验值（满级返回 0）
 */
public int calculateNextLevelExp(int level) {
    if (level >= MAX_LEVEL) return 0;
    return (int) Math.floor(BASE_EXP * Math.pow(EXP_MULTIPLIER, level) + EXP_LINEAR * level);
}
```

### 经验等级对照表（前 20 级）

| 等级 | 升级所需经验 | 等级 | 升级所需经验 |
|:---:|:---:|:---:|:---:|
| 1 | 100 | 11 | 403 |
| 2 | 115 | 12 | 459 |
| 3 | 132 | 13 | 522 |
| 4 | 152 | 14 | 593 |
| 5 | 175 | 15 | 671 |
| 6 | 201 | 16 | 759 |
| 7 | 231 | 17 | 857 |
| 8 | 265 | 18 | 965 |
| 9 | 303 | 19 | 1085 |
| 10 | 346 | 20 | 1218 |

> **说明**：采用指数+线性混合公式，前期升级快，后期逐渐放缓但不会过于陡峭。

## 三、升级逻辑

### 增加经验时的升级处理

```java
/**
 * 升级结果
 */
@Data
public class LevelUpResult {
    private int oldLevel;       // 升级前等级
    private int newLevel;       // 升级后等级
    private int levelsGained;   // 升级数
    private int pointsGained;   // 获得的属性点
    private int overflowExp;    // 溢出经验（新等级内的当前经验）
    private boolean isNewMaxLevel; // 是否达到满级
}

/**
 * 计算增加经验后的升级结果
 * @param currentLevel 当前等级
 * @param currentExp 当前等级内的经验值
 * @param expToAdd 要增加的经验值
 * @return 升级结果
 */
public LevelUpResult calculateLevelUp(int currentLevel, int currentExp, int expToAdd) {
    int level = currentLevel;
    int exp = currentExp + expToAdd;
    int levelsGained = 0;

    // 循环升级直到经验不足或满级
    while (level < MAX_LEVEL) {
        int nextExp = calculateNextLevelExp(level);
        if (exp < nextExp) break;

        exp -= nextExp;
        level++;
        levelsGained++;
    }

    int pointsGained = levelsGained * POINTS_PER_LEVEL;
    boolean isNewMaxLevel = level == MAX_LEVEL;

    return new LevelUpResult(
        currentLevel, level, levelsGained, pointsGained, exp, isNewMaxLevel
    );
}
```

## 四、数据库字段

### characters 表升级相关字段

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `level` | Integer | 当前等级（1-100） |
| `experience` | Integer | 当前等级内的经验值 |
| `available_points` | Integer | 可用自由属性点数 |
| `next_level_exp` | Integer | 下一级所需经验（前端展示用，可后端计算返回） |

> **说明**：`experience` 存储的是当前等级内的进度，而非累计经验。每次升级后重置为溢出值。

## 五、接口规范

### 1. 增加经验接口

**POST `/character/add-experience`**

**请求参数：**

```json
{
  "characterId": "uuid",
  "expToAdd": 500
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| `characterId` | `string` | ✅ | 角色 UUID |
| `expToAdd` | `number` | ✅ | 要增加的经验值（≥ 0） |

**成功响应：**

```json
{
  "code": 200,
  "message": "恭喜升级！获得 3 点属性点",
  "data": {
    "character": {
      "id": "uuid",
      "level": 16,
      "experience": 800,
      "nextLevelExp": 759,
      "availablePoints": 6,
      // ... 其他角色字段
    },
    "levelUp": {
      "oldLevel": 15,
      "newLevel": 16,
      "levelsGained": 1,
      "pointsGained": 3,
      "overflowExp": 800,
      "isNewMaxLevel": false
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `data.character` | 更新后的完整角色数据 |
| `data.levelUp` | 升级结果，未升级时为 `null` |

**失败响应：**

```json
{
  "code": 404,
  "message": "角色不存在",
  "data": null
}
```

```json
{
  "code": 400,
  "message": "经验值不能为负数",
  "data": null
}
```

## 七、升级处理完整流程

```java
/**
 * 增加角色经验并处理升级
 */
public AddExperienceResult addExperience(String characterId, int expToAdd) {
    // 1. 校验
    if (expToAdd < 0) {
        throw new BusinessException("经验值不能为负数");
    }

    Character character = characterRepository.findById(characterId);
    if (character == null) {
        throw new BusinessException("角色不存在");
    }

    if (character.getLevel() >= MAX_LEVEL) {
        // 满级不增加经验，但仍返回角色数据
        return new AddExperienceResult(character, null);
    }

    // 2. 计算升级结果
    LevelUpResult levelUp = calculateLevelUp(
        character.getLevel(),
        character.getExperience(),
        expToAdd
    );

    // 3. 更新角色数据
    character.setLevel(levelUp.getNewLevel());
    character.setExperience(levelUp.getOverflowExp());
    character.setAvailablePoints(character.getAvailablePoints() + levelUp.getPointsGained());
    character.setUpdateTime(new Date());

    // 4. 保存数据库
    characterRepository.update(character);

    // 5. 返回结果
    return new AddExperienceResult(character, levelUp);
}
```

## 八、注意事项

1. **满级处理**：达到 100 级后，`experience` 不再增长，`nextLevelExp` 返回 0
2. **溢出经验**：连续升级时，多余经验会累积到最终等级，需正确计算
3. **属性点累积**：连续升级获得的属性点应累加到 `available_points`
4. **前端动画**：前端需要 `levelUp` 结构体来触发升级庆祝动效
5. **并发安全**：增加经验和属性加点需保证并发安全（防止属性点复制漏洞）
6. **经验道具**：使用经验道具时调用同一接口，道具使用逻辑在调用前处理

## 九、前端动画触发时机

前端在收到包含 `levelUp` 的响应后：

1. 经验条填满动画（约 0.5s）
2. 经验条闪光效果
3. 升级粒子爆炸动效
4. 升级弹窗显示（等级变化、属性点奖励、解锁技能）

动效总时长约 2-3 秒，期间用户可点击跳过。
