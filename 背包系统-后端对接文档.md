# 背包系统 - 后端对接文档

> 版本：v1.0.0
> 更新日期：2026-04-30
> 说明：前后端需使用相同的物品分类与属性结构，确保数据一致性

---

## 一、物品分类

| 分类 | 英文名 | 说明 |
|:---|:---:|:---|
| 消耗品 | `consumable` | 药水、经验道具、复活道具等，使用后消失或减少数量 |
| 材料 | `material` | 副本产出材料、强化材料、进化材料等，用于合成/强化 |
| 装备 | `equipment` | 可穿戴的装备（武器/头盔/胸甲/护腿/饰品），详见装备系统文档 |

---

## 二、物品稀有度

与装备系统统一，使用四级稀有度：

| 稀有度 | 英文名 | 颜色标识 | 说明 |
|:---|:---:|:---:|:---|
| 普通 | `Normal` | 白色 | 基础物品 |
| 稀有 | `Rare` | 蓝色 | 较为少见 |
| 史诗 | `Epic` | 紫色 | 高级物品 |
| 传说 | `Legendary` | 橙色 | 顶级物品 |

## 三、数据库表结构

### 1. 物品静态配置表 (items)

存储所有物品的基础信息，前端展示用：

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `item_id` | INT (PK) | 物品模板 ID |
| `name` | VARCHAR(64) | 物品名称 |
| `category` | ENUM('consumable','material','equipment') | 物品分类 |
| `rarity` | ENUM('Normal','Rare','Epic','Legendary') | 稀有度 |
| `description` | VARCHAR(512) | 物品描述 |
| `icon_url` | VARCHAR(255) | 图标资源路径（可为 NULL） |
| `max_stack` | INT | 最大堆叠数量（装备=1，消耗品/材料可堆叠） |
| `sell_price` | INT | 出售价格（0 表示不可出售） |

**消耗品额外字段（仅 category='consumable'）：**

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `effect_type` | ENUM('heal_hp','heal_mp','add_exp','revive') | 效果类型 |
| `effect_value` | INT | 效果数值（如恢复 50 HP） |

**材料额外字段（仅 category='material'）：**

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `source` | VARCHAR(255) | 获取途径描述 |
| `usage` | VARCHAR(255) | 用途描述 |

> **说明**：装备类物品的详细属性由装备系统单独管理，详见《装备系统-后端对接文档》。

### 2. 角色背包表 (character_inventory)

存储角色拥有的物品实例：

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `id` | UUID (PK) | 背包记录唯一标识 |
| `character_id` | UUID (FK → characters) | 所属角色 |
| `item_id` | INT (FK → items) | 物品模板 ID |
| `quantity` | INT | 持有数量（≥1） |
| `obtained_at` | TIMESTAMP | 获取时间 |

**约束**：
- 同一角色 + 同一物品模板，最多 1 条记录（堆叠合并）
- `quantity` ≤ `items.max_stack`
- 背包容量上限：32 种物品（可扩展）

---

## 四、堆叠规则

1. **可堆叠物品**：消耗品、材料（`max_stack > 1`）
   - 获取同类物品时，若已有该物品且未达上限，增加 `quantity`
   - 若已达上限或不存在，新建记录

2. **不可堆叠物品**：装备（`max_stack = 1`）
   - 每件装备独立一条记录
   - 即使是相同装备模板，也不合并

3. **堆叠上限**：
   - 消耗品：通常 99 或 20
   - 材料：通常 99 或 50
   - 装备：固定为 1

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |

## 六、物品效果类型说明

消耗品效果类型及处理逻辑：

| effect_type | 说明 | 使用逻辑 |
|:---|:---|:---|
| `heal_hp` | 恢复生命值 | `character.hp = min(character.hp + value, character.maxHp)` |
| `heal_mp` | 恢复魔法值 | `character.mp = min(character.mp + value, character.maxMp)` |
| `add_exp` | 增加经验值 | 调用经验增加接口，可能触发升级 |
| `revive` | 复活角色 | 仅在角色死亡状态可用，恢复一定 HP |

## 九、物品图标资源规范

### 图标 URL 字段

`iconUrl` 字段存储物品图标资源路径，格式：

```
/assets/items/{category}/{itemId}.png
```

| 分类 | 目录 |
|:---|:---|
| 消耗品 | `/assets/items/consumable/` |
| 材料 | `/assets/items/material/` |
| 装备 | `/assets/items/equipment/` |

**命名示例：**

| itemId | name | iconUrl |
|:---:|:---|:---|
| 1001 | 小型生命药水 | `/assets/items/consumable/1001.png` |
| 2001 | 铁矿石 | `/assets/items/material/2001.png` |

> **前期方案**：无美术资源时 `iconUrl` 返回 `null`，前端使用 Lucide 分类图标占位：
> - 消耗品：Droplet（药滴）
> - 材料：Gem（宝石）
> - 装备：Shirt（衣物）

### 图标尺寸建议

| 场景 | 尺寸 |
|:---|:---|
| 背包格子图标 | 32×32 px |
| 物品详情弹窗图标 | 48×48 px |
| 悬浮提示图标 | 24×24 px |

## 十、品质颜色渲染规范（前端）

前端使用以下颜色渲染物品品质，后端无需处理，仅需了解：

| 稀有度 | 边框颜色 | 底纹渐变 | 图标颜色 |
|:---|:---:|:---|:---:|
| 普通 | `#6e6e73` | 8% 透明度灰色 | `#6e6e73` |
| 稀有 | `#0071e3` | 18%→8% 蓝色渐变 | `#0071e3` |
| 史诗 | `#af52de` | 18%→8% 紫色渐变 | `#af52de` |
| 传说 | `#ff9500` | 18%→8% 橙色渐变 | `#ff9500` |

**渲染效果：**

- 边框：2px 实线，颜色对应稀有度
- 底纹：`linear-gradient(135deg, {color}18 0%, {color}08 100%)`
- 图标：分类图标，颜色对应稀有度
- 悬浮提示：物品名称使用稀有度颜色高亮

## 十一、注意事项

1. **堆叠合并**：获取物品时需判断是否已存在同类物品，执行合并逻辑
2. **数量校验**：使用/丢弃物品时校验 `quantity`，不能为负数
3. **背包容量**：当前上限 32 种物品，超出时需提示玩家整理或扩展
4. **物品唯一性**：`itemId` 是全局唯一标识，跨角色共享静态配置
5. **图标资源**：前期无美术资源时 `iconUrl` 返回 `null`，前端使用分类图标占位
6. **品质渲染**：前端统一处理品质颜色，后端仅需返回 `rarity` 字段
