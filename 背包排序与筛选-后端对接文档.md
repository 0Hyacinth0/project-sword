# 背包排序与筛选 - 后端对接文档

> 版本：v1.0.0
> 更新日期：2026-05-01
> 说明：背包排序与筛选为纯前端功能，后端接口无需改动

---

## 一、功能概述

背包排序与筛选功能完全在前端实现，基于后端返回的完整背包物品列表进行本地处理。

**后端无需新增接口或修改现有接口。**

---

## 二、前端排序功能

### 2.1 排序字段

| 排序字段 | 字段名 | 说明 |
|:---|:---|:---|
| 获取时间 | `obtainedAt` | 按物品获取时间排序（默认） |
| 名称 | `name` | 按物品名称拼音排序 |
| 稀有度 | `rarity` | 按稀有度权重排序（普通→稀有→史诗→传说） |
| 数量 | `quantity` | 按持有数量排序 |
| 售价 | `sellPrice` | 按出售价格排序 |

### 2.2 稀有度权重

| 稀有度 | 权重 |
|:---|:---:|
| Normal | 0 |
| Rare | 1 |
| Epic | 2 |
| Legendary | 3 |

### 2.3 排序方向

- **升序（asc）**：数值从小到大、名称 A→Z、时间从早到晚
- **降序（desc）**：数值从大到小、名称 Z→A、时间从晚到早（默认）

### 2.4 交互逻辑

- 点击排序按钮切换排序字段（循环切换）
- 点击同一排序字段时切换升降序
- 排序图标旁显示升降序箭头

---

## 三、前端筛选功能

### 3.1 分类筛选（已有）

| 标签 | 筛选值 | 说明 |
|:---|:---|:---|
| 全部 | `all` | 显示所有物品 |
| 消耗品 | `consumable` | 仅显示消耗品 |
| 材料 | `material` | 仅显示材料 |
| 装备 | `equipment` | 仅显示装备 |

### 3.2 稀有度筛选

| 选项 | 筛选值 | 说明 |
|:---|:---|:---|
| 全部品质 | `all` | 不按稀有度过滤（默认） |
| 普通 | `Normal` | 仅显示普通物品 |
| 稀有 | `Rare` | 仅显示稀有物品 |
| 史诗 | `Epic` | 仅显示史诗物品 |
| 传说 | `Legendary` | 仅显示传说物品 |

### 3.3 名称搜索

- 输入关键词后实时过滤，匹配物品名称（不区分大小写）
- 空输入时不筛选
- 搜索与分类筛选、稀有度筛选同时生效

---

## 四、筛选组合逻辑

三个筛选维度同时生效，取交集：

```
最终结果 = 分类筛选 ∩ 稀有度筛选 ∩ 名称搜索
排序应用在筛选结果之上
```

---

## 五、后端接口依赖

仅依赖已有接口：

| 接口 | 方法 | 说明 |
|:---|:---:|:---|
| `/inventory/list/{characterId}` | GET | 获取完整背包物品列表 |

**后端返回完整的背包数据，前端在本地进行排序和筛选。** 后端无需实现服务端排序/分页。
