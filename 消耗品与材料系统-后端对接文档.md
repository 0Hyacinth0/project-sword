## 一、消耗品类

### 1.1 消耗品效果类型

| effect_type | 说明 | 使用逻辑 |
|:---|:---|:---|
| `heal_hp` | 恢复生命值 | `character.hp = min(character.hp + value × quantity, character.maxHp)` |
| `heal_mp` | 恢复魔法值 | `character.mp = min(character.mp + value × quantity, character.maxMp)` |
| `add_exp` | 增加经验值 | 调用经验增加接口，可能触发升级 |
| `revive` | 复活角色 | 仅在角色死亡状态可用，恢复 `value%` 最大生命值 |

### 1.3 使用限制

| 效果类型 | 限制条件 | 错误提示 |
|:---|:---|:---|
| heal_hp | HP 已满（`hp == maxHp`） | "HP 已满，无需恢复" |
| heal_mp | MP 已满（`mp == maxMp`） | "MP 已满，无需恢复" |
| add_exp | 无特殊限制 | — |
| revive | 角色未死亡 | "角色未阵亡，无法使用" |

## 三、数据库表结构

复用已有的 `items` 表和 `character_inventory` 表（见《背包系统-后端对接文档》）。

消耗品额外字段需存入 `items` 表：
```
effect_type  VARCHAR(32)   -- 效果类型（heal_hp / heal_mp / add_exp / revive）
effect_value INT           -- 效果数值
```

材料额外字段需存入 `items` 表：
```
source       VARCHAR(255)  -- 获取途径描述
usage        VARCHAR(255)  -- 用途描述
```

## 四、前端展示规范

### 4.1 物品描述弹窗

弹窗统一展示以下信息：

| 区域 | 内容 | 适用范围 |
|:---|:---|:---|
| 头部 | 图标 + 名称（稀有度颜色）+ 稀有度·分类 | 所有物品 |
| 描述 | 物品描述文本 | 所有物品 |
| 效果 | 效果描述（如"恢复 150 HP"） | 仅消耗品 |
| 来源 | 获取途径 | 仅材料 |
| 用途 | 材料用途 | 仅材料 |
| 底部 | 数量/堆叠上限 + 售价 | 所有物品 |
| 操作 | 使用按钮（消耗品）/ 丢弃按钮（所有） | 所有物品 |
