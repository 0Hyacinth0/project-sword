> 版本：v1.0.0
> 更新日期：2026-05-01
> 说明：物品使用、丢弃接口规范及堆叠规则

---

## 一、接口规范

### 1. 使用消耗品

**POST `/inventory/use`**

**请求参数：**

```json
{
  "characterId": "a1b2c3d4-...",
  "inventoryId": "inv-001",
  "quantity": 1
}
```

| 参数 | 类型 | 必填 | 说明 |
|:---|:---:|:---:|:---|
| `characterId` | string | ✅ | 角色 UUID |
| `inventoryId` | string | ✅ | 背包记录唯一 ID |
| `quantity` | int | ✅ | 使用数量（≥1） |

**成功响应：**

```json
{
  "code": 200,
  "message": "使用成功",
  "data": {
    "effects": ["恢复 50 HP"],
    "message": "成功使用 小型生命药水×1，恢复 50 点生命值"
  }
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `effects` | string[] | 效果描述列表，前端用于逐条展示 |
| `message` | string | 汇总提示，前端用于 Toast 展示 |

**失败响应：**

```json
{ "code": 400, "message": "该物品无法使用", "data": null }
{ "code": 400, "message": "数量不足", "data": null }
{ "code": 400, "message": "HP 已满，无需恢复", "data": null }
{ "code": 400, "message": "MP 已满，无需恢复", "data": null }
{ "code": 404, "message": "物品不存在", "data": null }
```

**后端处理逻辑：**

1. 校验 `inventoryId` 是否存在且属于 `characterId`
2. 校验物品类型：仅 `consumable` 可使用（`category != 'consumable'` → 返回 400）
3. 校验 `quantity`：1 ≤ quantity ≤ 当前持有数量
4. 执行效果（根据 `effect_type`）：
   - `heal_hp`：`hp = min(hp + value × quantity, maxHp)`，若 `hp == maxHp` → 返回 "HP 已满"
   - `heal_mp`：`mp = min(mp + value × quantity, maxMp)`，若 `mp == maxMp` → 返回 "MP 已满"
   - `add_exp`：调用经验增加接口，可能触发升级
   - `revive`：仅在死亡状态可用，恢复一定 HP
5. 扣减数量：`quantity -= 使用数量`
6. 若扣减后数量 ≤ 0，删除该背包记录
7. 返回效果描述和汇总消息

### 2. 丢弃物品

**POST `/inventory/discard`**

**请求参数：**

```json
{
  "characterId": "a1b2c3d4-...",
  "inventoryId": "inv-001",
  "quantity": 5
}
```

| 参数 | 类型 | 必填 | 说明 |
|:---|:---:|:---:|:---|
| `characterId` | string | ✅ | 角色 UUID |
| `inventoryId` | string | ✅ | 背包记录唯一 ID |
| `quantity` | int | ✅ | 丢弃数量（≥1） |

**成功响应：**

```json
{ "code": 200, "message": "丢弃成功", "data": null }
```

**失败响应：**

```json
{ "code": 400, "message": "数量不足", "data": null }
{ "code": 404, "message": "物品不存在", "data": null }
{ "code": 403, "message": "无权操作该角色背包", "data": null }
```

**后端处理逻辑：**

1. 校验 `inventoryId` 是否存在且属于 `characterId`
2. 校验 `quantity`：1 ≤ quantity ≤ 当前持有数量
3. 扣减数量：`quantity -= 丢弃数量`
4. 若扣减后数量 ≤ 0，删除该背包记录
5. 返回成功

> **注意**：丢弃操作不可撤销，后端无需回收机制，直接从 `character_inventory` 表扣减或删除。

## 二、堆叠规则（后端逻辑）

堆叠在**获取物品**时由后端自动处理，前端无需额外调用接口。

### 获取物品时的堆叠合并逻辑

```
当角色获得物品 (itemId, quantity) 时：
1. 查询 character_inventory 中该角色是否已有该 itemId 的记录
2. 若存在：
   a. 当前数量 + 新数量 ≤ max_stack → 直接增加 quantity
   b. 当前数量 + 新数量 > max_stack → 增加到 max_stack，剩余部分新建记录（或返回溢出提示）
3. 若不存在：
   a. 背包物品种类 < 32（容量上限）→ 新建记录
   b. 背包物品种类 ≥ 32 → 返回 "背包已满" 错误
```

### 装备类物品特殊处理

- 装备类物品 `max_stack = 1`，每件装备独立一条记录，不合并
- 即使是相同装备模板（如两把 "铁剑"），也各自独立

### 堆叠上限参考

| 物品类型 | 默认 max_stack | 说明 |
|:---|:---:|:---|
| 消耗品 | 99 或 20 | 药水类 99，卷轴类 10~20 |
| 材料 | 99 或 50 | 基础材料 99，稀有材料 10~50 |
| 装备 | 1 | 固定不可堆叠 |

## 三、错误码汇总

| code | 场景 | 说明 |
|:---:|:---|:---|
| 200 | 成功 | 操作成功 |
| 400 | 参数错误 | 数量 ≤ 0、类型不支持等 |
| 400 | 数量不足 | 请求的数量 > 当前持有数量 |
| 400 | HP/MP 已满 | 恢复类消耗品在满血/满蓝时不可使用 |
| 400 | 背包已满 | 获取物品时背包种类已达上限（32） |
| 403 | 无权限 | 操作非自己角色的背包 |
| 404 | 物品不存在 | inventoryId 无效或已删除 |

## 四、前端交互流程

### 使用消耗品

1. 用户点击背包格子 → 弹出物品详情弹窗
2. 弹窗底部显示「使用」按钮（仅消耗品）
3. 点击「使用」→ 展开数量选择器（- / 数字 / +）
4. 确认数量后点击「确认」
5. 前端调用 `POST /inventory/use`
6. 成功 → Toast 提示效果、刷新背包、关闭弹窗
7. 失败 → Toast 提示错误信息

### 丢弃物品

1. 用户点击背包格子 → 弹出物品详情弹窗
2. 弹窗底部显示「丢弃」按钮
3. 点击「丢弃」→ 展开数量选择器
4. 确认数量后点击「确认」→ 弹出二次确认
5. 二次确认后前端调用 `POST /inventory/discard`
6. 成功 → Toast 提示、刷新背包、关闭弹窗
7. 失败 → Toast 提示错误信息

## 五、数据库变更说明

本次新增接口不涉及数据库表结构变更，复用已有的 `character_inventory` 表和 `items` 表。

操作仅涉及：
- `UPDATE character_inventory SET quantity = quantity - ? WHERE id = ?`
- `DELETE FROM character_inventory WHERE id = ? AND quantity <= 0`
