## 2. 属性加点

### 请求

**方法**: `POST`

**路径**: `/character/update-attributes`

**请求头**:
```
X-Access-Token: {JWT_TOKEN}
Content-Type: application/json
```

**请求体**:
```json
{
  "characterId": "uuid",
  "str": 2,
  "int": 0,
  "agi": 1
}
```

**请求参数说明**:
| 参数名 | 类型 | 必填 | 说明 |
|:---|:---:|:---:|:---|
| characterId | UUID | 是 | 角色 ID |
| str | Number | 是 | 力量增加值（非负整数） |
| int | Number | 是 | 智力增加值（非负整数） |
| agi | Number | 是 | 敏捷增加值（非负整数） |

### 校验规则

1. **可用点数校验**: `str + int + agi` 不能超过角色的 `availablePoints`
2. **非负校验**: 各属性增加值必须为非负整数
3. **衍生属性重算**: 加点后需重新计算以下衍生属性：
   - `maxHp = baseHp + strength * 5 + equipment_bonus`
   - `maxMp = baseMp + intelligence * 5 + equipment_bonus`
   - `physicalAttack = baseAttack + strength * 2 + equipment_bonus`
   - `magicAttack = baseAttack + intelligence * 2 + equipment_bonus`
   - `dodgeRate = baseDodge + agility * 0.01 + equipment_bonus`
   - `criticalRate = baseCrit + agility * 0.008 + equipment_bonus`

### 响应

**成功响应**:
```json
{
  "code": 200,
  "message": "属性加点成功",
  "data": {
    "id": "uuid",
    "strength": 12,
    "intelligence": 3,
    "agility": 6,
    "availablePoints": 0,
    "maxHp": 160,
    "maxMp": 35,
    "physicalAttack": 24,
    "magicAttack": 6,
    "dodgeRate": 0.06,
    "criticalRate": 0.048,
    "hp": 160,
    "mp": 35
  }
}
```

**失败响应示例**:

*可用点数不足*:
```json
{
  "code": 400,
  "message": "可用属性点不足",
  "data": null
}
```

*角色不存在*:
```json
{
  "code": 404,
  "message": "角色不存在",
  "data": null
}
```

## 5. 前端注意事项

1. **字段命名转换**: 后端使用 snake_case，jeecg-boot 会自动转换为 camelCase，前端接口定义使用 camelCase

2. **百分比属性**: `dodgeRate` 和 `criticalRate` 返回的是 0-1 的小数，前端显示时需要乘以 100 并格式化为百分比

3. **空槽位**: 装备槽位为空时返回 `null`，前端需要显示空槽位样式

4. **战宠经验**: 战宠的 `exp` 和 `maxExp` 用于显示经验进度条

5. **Mock 开关**: 前端代码中每个接口都有独立的 Mock 开关，联调完成后需要关闭 Mock：
   ```typescript
   const MOCK = {
     list: false,
     create: false,
     delete: false,
     checkName: false,
     info: false,
     updateAttributes: false  // ← 联调后改为 false
   }
   ```
